# Настройка хранения файлов в Coolify

## Проблема
После загрузки файлов в рассылке возникала ошибка 403 при переходе по ссылке на файл.

## Решение
Были внесены изменения в `nixpacks.toml` для правильной настройки хранения файлов:

### 1. Создание символической ссылки
Добавлена команда `php artisan storage:link` в процесс сборки и запуска, чтобы создать символическую ссылку `public/storage` → `storage/app/public`.

### 2. Настройка прав доступа
Установлены правильные права доступа для директории `storage/app/public`:
- Права: `775`
- Владелец: `www-data:www-data`

### 3. Конфигурация Nginx
Добавлена специальная секция `location /storage` в конфигурацию Nginx для прямой отдачи файлов без прохождения через PHP:
- Файлы отдаются напрямую из `public/storage`
- Установлен кэш на 30 дней
- Отключено логирование для статических файлов

### 4. Создание директорий
Автоматически создается директория `storage/app/public/newsletters/media` для загруженных файлов рассылок.

## Что нужно сделать после деплоя

1. **Проверить символическую ссылку:**
   ```bash
   ls -la /app/public/storage
   ```
   Должна быть ссылка на `../storage/app/public`

2. **Проверить права доступа:**
   ```bash
   ls -la /app/storage/app/public
   ```
   Владелец должен быть `www-data:www-data`, права `775`

3. **Проверить доступность файлов:**
   После загрузки файла в рассылке, проверьте доступность по URL:
   ```
   https://your-domain.com/storage/newsletters/media/filename.ext
   ```

## Ручная настройка (если автоматическая не сработала)

Если после деплоя проблема сохраняется, выполните в контейнере:

```bash
# Создать символическую ссылку
php artisan storage:link

# Установить права доступа
chmod -R 775 storage/app/public
chown -R www-data:www-data storage/app/public

# Перезапустить Nginx (если нужно)
supervisorctl restart worker-nginx
```

## Проверка работы

1. Загрузите файл в рассылке через админ-панель
2. Проверьте, что файл появился в `storage/app/public/newsletters/media/`
3. Откройте ссылку на файл в браузере - должна открываться без ошибки 403

## Дополнительная информация

- Файлы хранятся в: `storage/app/public/newsletters/media/`
- URL файлов: `https://your-domain.com/storage/newsletters/media/filename.ext`
- Максимальный размер файла: 10MB (настраивается в `MailingListController.php`)
