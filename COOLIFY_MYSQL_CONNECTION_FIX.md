# Решение проблемы подключения к MySQL в Coolify

## Проблема
При попытке подключения к MySQL контейнеру возникает ошибка:
```
mysqli::real_connect(): php_network_getaddresses: getaddrinfo for agsk0os4cw0sk04sogcsogk0 failed: Name does not resolve
```

## Причина
В Coolify при использовании Standalone Docker контейнеров для подключения к MySQL может возникнуть проблема с DNS разрешением имен. Это может быть связано с:
- Контейнеры не находятся в одной Docker сети
- DNS в Docker сети не работает корректно
- Имя контейнера не разрешается через Docker DNS

## Решение

### Вариант 1: Использовать имя сервиса (Рекомендуется)

1. **Найдите имя сервиса MySQL в Coolify:**
   - Откройте настройки MySQL контейнера в Coolify
   - Найдите поле "Service Name" или "Container Name" (без суффикса с ID)
   - Обычно это что-то вроде `mysql-database` или просто `mysql`

2. **Обновите переменную окружения `DB_HOST` в вашем приложении:**
   - В Coolify откройте настройки вашего приложения
   - Найдите переменные окружения (Environment Variables)
   - Измените `DB_HOST` на имя сервиса MySQL (например, `mysql-database`)
   - Сохраните и перезапустите приложение

### Вариант 2: Проверить сеть Docker

1. **Убедитесь, что оба контейнера в одной сети:**
   - В Coolify проверьте настройки MySQL контейнера
   - Убедитесь, что Network указана как `coolify` (или та же сеть, что у приложения)
   - Проверьте настройки вашего приложения - Network должна совпадать

2. **Проверьте имя сети:**
   - В настройках MySQL: Network должна быть `coolify` (Standalone Docker (coolify) Network: coolify)
   - В настройках приложения: Network должна быть такой же

### Вариант 3: Использовать внутренний IP (РЕКОМЕНДУЕТСЯ, если DNS не работает)

Если имя сервиса не работает, **используйте внутренний IP адрес MySQL контейнера**:

1. **Найдите IP адрес MySQL контейнера через Coolify:**
   - Откройте MySQL контейнер в Coolify
   - Перейдите в раздел "Logs" или "Terminal"
   - Выполните команду: `hostname -I` или `ip addr show`
   - Или в настройках контейнера найдите "IP Address"

2. **Альтернативный способ - через Docker на сервере:**
   ```bash
   # На сервере, где запущен Coolify, выполните:
   docker network inspect coolify
   # Найдите секцию с вашим MySQL контейнером и его IP адрес
   ```

3. **Используйте IP в DB_HOST:**
   - В настройках вашего приложения в Coolify
   - Перейдите в "Environment Variables"
   - Измените `DB_HOST` на IP адрес (например: `DB_HOST=172.18.0.5`)
   - **ВАЖНО:** Также измените `DB_HOST` в настройках phpMyAdmin контейнера
   - Сохраните и перезапустите оба контейнера

4. **⚠️ Внимание:** 
   - IP может измениться при перезапуске контейнера
   - Для постоянного решения лучше настроить статический IP или исправить проблему с сетью

### Вариант 4: Использовать имя контейнера без суффикса

В некоторых случаях Coolify создает алиасы. Попробуйте использовать имя без случайного суффикса:
- Вместо: `mysql-database-agsk0os4cw0sk04sogcsogk0`
- Попробуйте: `mysql-database`

## Пошаговая инструкция (Рекомендуемый способ)

1. **В Coolify откройте MySQL контейнер:**
   - Перейдите в раздел "Resources" → "Standalone Docker"
   - Найдите ваш MySQL контейнер
   - Откройте его настройки

2. **Найдите имя сервиса:**
   - В настройках контейнера найдите поле "Name" или "Service Name"
   - Это имя без случайного суффикса (ID)
   - Например: если контейнер называется `mysql-database-agsk0os4cw0sk04sogcsogk0`, то имя сервиса может быть `mysql-database`

3. **Обновите переменные окружения приложения:**
   - Откройте настройки вашего приложения в Coolify
   - Перейдите в раздел "Environment Variables"
   - Найдите `DB_HOST` и измените на имя сервиса из шага 2
   - Пример: `DB_HOST=mysql-database`
   - Сохраните изменения

4. **Перезапустите приложение:**
   - В Coolify нажмите "Redeploy" или перезапустите контейнер приложения
   - Проверьте логи на наличие ошибок подключения

5. **Проверьте подключение:**
   - Откройте логи приложения
   - Убедитесь, что ошибка `Name does not resolve` больше не появляется
   - Проверьте работу приложения

## Детальная диагностика проблемы

### Шаг 1: Проверка сети Docker

Выполните на сервере, где запущен Coolify:

```bash
# Проверьте, какие контейнеры в сети coolify
docker network inspect coolify

# Найдите ваш MySQL контейнер и его IP адрес
docker ps | grep mysql

# Проверьте IP адрес конкретного контейнера
docker inspect mysql-database-agsk0os4cw0sk04sogcsogk0 | grep IPAddress
```

### Шаг 2: Проверка DNS в контейнере приложения

В терминале контейнера приложения (через Coolify → Terminal):

```bash
# Проверьте /etc/hosts
cat /etc/hosts

# Попробуйте разрешить имя
nslookup mysql-database-agsk0os4cw0sk04sogcsogk0
# или
getent hosts mysql-database-agsk0os4cw0sk04sogcsogk0

# Попробуйте пинг
ping -c 3 mysql-database-agsk0os4cw0sk04sogcsogk0
```

### Шаг 3: Проверка доступности MySQL по IP

```bash
# В контейнере приложения попробуйте подключиться по IP
# (замените IP на реальный адрес MySQL контейнера)
telnet 172.18.0.5 3306
# или
nc -zv 172.18.0.5 3306
```

### Шаг 4: Проверка настроек сети в Coolify

1. **MySQL контейнер:**
   - Убедитесь, что Network = `coolify`
   - Проверьте, что контейнер запущен и работает

2. **Приложение контейнер:**
   - Убедитесь, что Network = `coolify` (та же сеть!)
   - Проверьте, что контейнер запущен

3. **phpMyAdmin контейнер:**
   - Убедитесь, что Network = `coolify` (та же сеть!)
   - Проверьте переменные окружения `PMA_HOST`

## Решение через IP адрес (если DNS не работает)

### Для Laravel приложения:

1. **Найдите IP адрес MySQL:**
   ```bash
   docker inspect mysql-database-agsk0os4cw0sk04sogcsogk0 | grep -A 20 "Networks"
   ```

2. **В Coolify обновите переменные окружения приложения:**
   ```
   DB_HOST=172.18.0.5  # Замените на реальный IP
   DB_PORT=3306
   ```

3. **Перезапустите приложение**

### Для phpMyAdmin:

1. **В настройках phpMyAdmin контейнера в Coolify:**
   - Найдите переменные окружения
   - Установите:
     ```
     PMA_HOST=172.18.0.5  # Замените на реальный IP MySQL
     PMA_PORT=3306
     ```
   - Или если используется `PMA_ARBITRARY=1`, убедитесь, что в интерфейсе phpMyAdmin указан IP адрес

2. **Перезапустите phpMyAdmin контейнер**

## Альтернативное решение: Использование host.docker.internal

Если контейнеры не могут видеть друг друга через Docker сеть, попробуйте:

```env
DB_HOST=host.docker.internal
```

Но это работает только если MySQL доступен на хосте, а не в контейнере.

## Проверка порта и доступности

1. **Убедитесь, что MySQL слушает на правильном порту:**
   ```bash
   # В MySQL контейнере
   netstat -tlnp | grep 3306
   ```

2. **Проверьте, что порт не заблокирован:**
   - В настройках MySQL контейнера убедитесь, что порт 3306 открыт
   - Проверьте firewall на сервере

## Пример правильной конфигурации

В переменных окружения приложения должны быть установлены:

```env
DB_CONNECTION=mysql
DB_HOST=mysql-database          # Имя сервиса, не полное имя контейнера
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

## Быстрое решение через IP адрес (РЕКОМЕНДУЕТСЯ)

Если DNS не работает, используйте IP адрес MySQL контейнера:

### Шаг 1: Найдите IP адрес MySQL контейнера

**Способ 1 - через Coolify:**
1. Откройте MySQL контейнер в Coolify
2. Перейдите в раздел "Terminal" или "Logs"
3. Выполните команду: `hostname -I` или `ip addr show eth0`
4. Запишите IP адрес (например: `172.18.0.5`)

**Способ 2 - через сервер (SSH):**
```bash
# На сервере, где запущен Coolify
docker inspect mysql-database-agsk0os4cw0sk04sogcsogk0 | grep -i ipaddress

# Или более детально:
docker inspect mysql-database-agsk0os4cw0sk04sogcsogk0 | grep -A 10 "Networks"
```

**Способ 3 - через docker network:**
```bash
docker network inspect coolify | grep -A 10 "mysql-database"
```

### Шаг 2: Настройте Laravel приложение

1. В Coolify откройте настройки вашего приложения
2. Перейдите в раздел "Environment Variables"
3. Найдите `DB_HOST` и измените на IP адрес:
   ```
   DB_HOST=172.18.0.5  # Замените на ваш реальный IP
   ```
4. Сохраните изменения
5. Перезапустите приложение (Redeploy)

### Шаг 3: Настройте phpMyAdmin

1. В Coolify откройте настройки phpMyAdmin контейнера
2. Перейдите в раздел "Environment Variables"
3. Найдите или добавьте переменные:
   ```
   PMA_HOST=172.18.0.5  # Замените на ваш реальный IP MySQL
   PMA_PORT=3306
   PMA_ARBITRARY=0  # Отключите произвольный сервер, если установлен
   ```
4. Сохраните изменения
5. Перезапустите phpMyAdmin контейнер

### Шаг 4: Альтернатива для phpMyAdmin (если PMA_HOST не работает)

Если переменная `PMA_HOST` не работает:

1. Откройте phpMyAdmin в браузере
2. В форме входа вручную укажите:
   - **Server:** `172.18.0.5` (IP адрес MySQL)
   - **Username:** ваш пользователь MySQL
   - **Password:** ваш пароль MySQL
3. Нажмите "Войти"

### Шаг 5: Проверка подключения

1. **Проверьте Laravel приложение:**
   - Откройте логи приложения в Coolify
   - Убедитесь, что нет ошибок подключения к БД
   - Попробуйте выполнить миграции: `php artisan migrate:status`

2. **Проверьте phpMyAdmin:**
   - Откройте phpMyAdmin в браузере
   - Убедитесь, что подключение работает
   - Проверьте список баз данных

## Примечания

- **Если DNS не работает, используйте IP адрес** - это самое надежное решение
- Убедитесь, что **все три контейнера** (приложение, MySQL, phpMyAdmin) находятся в **одной сети** `coolify`
- После изменения переменных окружения **обязательно перезапустите контейнеры**
- IP адрес может измениться при перезапуске контейнера, но в Coolify обычно он стабилен
- Для постоянного решения проблемы с DNS можно настроить статический IP или использовать docker-compose с явными именами сервисов
