Платежный шлюз для проведения онлайн оплаты на стороннем сервере.

== Description ==

Плагин создает новый тип оплаты "Онлайн оплата" для WooCommerce. При создании заказа плагин отправляет запрос на сторонний сервер с информацией по заказу (стоимость заказа, данные покупателя, номер заказа), получает ссылку на оплату и сохраняет её в мета поле заказа. Предусмотрен роут для хука оповещения об успешной/неуспешной оплате и последующем переводе статуса заказа на "Принят".

**Основные возможности:**

*   **Интеграция с внешним API:** Отправка данных заказа на сторонний сервер в формате JSON
*   **Получение ссылки на оплату:** Автоматическое получение и сохранение ссылки на оплату
*   **Обработка callback:** Автоматическая обработка уведомлений об успешной/неуспешной оплате
*   **Обновление статуса заказа:** Автоматический перевод заказа в статус "Принят" после успешной оплаты
*   **Полная интеграция с WooCommerce:** Корректная обработка статусов заказов и очистка корзины после успешной оплаты
*   **HPOS Compatible:** Плагин полностью совместим с High-Performance Order Storage
*   **Простая настройка:** Удобная страница настроек в админ-панели WooCommerce
*   **Логирование:** Подробное логирование всех операций для отладки

== Installation ==

1. Загрузите папку плагина в директорию `/wp-content/plugins/`
2. Активируйте плагин через меню 'Плагины' в WordPress
3. Перейдите в WooCommerce > Настройки > Оплата > Онлайн оплата
4. Настройте параметры подключения к стороннему серверу
5. Сохраните настройки

== Frequently Asked Questions ==

= Как настроить плагин? =

Перейдите в WooCommerce > Настройки > Оплата > Онлайн оплата и укажите URL API сервера и другие необходимые параметры.

= Какой формат данных отправляется на сторонний сервер? =

Плагин отправляет POST запрос в формате JSON со следующими данными:
- order_id: номер заказа
- order_total: стоимость заказа
- customer_name: имя покупателя
- customer_email: email покупателя
- customer_phone: телефон покупателя
- order_key: ключ заказа

= Какой формат ответа ожидается от сервера? =

Сервер должен вернуть JSON с полем `payment_url` или `url`, содержащим ссылку на страницу оплаты.

= Как обрабатываются callback-уведомления? =

Плагин ожидает POST запрос в формате JSON на эндпоинт `/wc-api/online_remote/` с данными о статусе оплаты.
