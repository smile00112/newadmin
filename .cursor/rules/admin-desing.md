# Admin UI Style Guide

Это правило описывает визуальный стиль админ‑панели Bagisto в этом проекте и предназначено как **единственный источник правды** для нейросетевых редакторов кода.

## Источники стиля

Основные файлы, определяющие внешний вид админки:

- `packages/Webkul/Admin/tailwind.config.js`
- `packages/Webkul/Admin/src/Resources/assets/css/app.css`
- `packages/Webkul/Admin/src/Resources/views/components/layouts/index.blade.php`
- `packages/Webkul/Admin/src/Resources/views/components/layouts/header/index.blade.php`
- (для AI‑виджета) `packages/Webkul/Admin/src/Resources/views/components/layouts/ai-assistant-chat.blade.php`

Нейросетевой редактор должен **ориентироваться на это правило** и использовать описанные ниже токены/паттерны при генерации нового интерфейса.

---

## Базовая технология

- UI админки построен на **Tailwind CSS** + кастомные классы в `app.css`.
- Вёрстка использует utility‑классы Tailwind и готовые CSS‑компоненты (`.primary-button`, `.box-shadow` и т.д.).
- Цвета, тени, радиусы и анимации заданы в `tailwind.config.js` в секции `theme.extend`.

---

## Цветовая палитра

Основные цвета и акценты:

- **Основной фиолетовый (primary/violet)**  
  Из `tailwind.config.js` (`colors.primary`, `colors.violet`):
  - Диапазон 50–900: от `#f5f3ff` до `#4c1d95`.
  - Используются для:
    - фоновых градиентов кнопок и карточек,
    - состояний наведения и фокуса,
    - акцентного текста (`.gradient-text`).

- **Accent**  
  `colors.accent` (розово‑фиолетовые оттенки) — дополнительные акценты, реже основного primary.

- **Поверхности (surface)**  
  `colors.surface` — светлые оттенки серого под фоном карточек, элементов UI.

- **Дополнительные цвета**  
  - `darkGreen`, `darkBlue`, `darkPink` — могут использоваться как брендовые/отдельные акценты.

**Рекомендация для генерации UI:**

- Для primary‑действий использовать фиолетово‑индиговые градиенты.
- Для success — зелёные градиенты.
- Для danger — розово‑красные.
- Фон и контейнеры — светло‑серые/белые поверхности с мягкими тенями.

## Типографика

- Основной шрифт админки: **`Poppins`, sans-serif**  
  Подключается в layout через Google Fonts и используется в Tailwind (`fontFamily.poppins`).
- Для иконок используется шрифт `bagisto-admin` и классы `icon-*` (см. `app.css`).

**Рекомендация:**

- Текст по умолчанию — размер около 14px (`p { @apply text-[14px]; }`).
- Использовать `font-semibold`/`font-bold` для заголовков и важных элементов.

## Кнопки (button styles)

Использовать уже имеющиеся CSS‑классы:

- **Primary кнопка** — `.primary-button`  
  - Градиент `from-indigo-600 to-indigo-700`
  - Белый текст, скругление `rounded-xl`
  - Тень и hover‑эффект (усиление градиента и тени, лёгкий `-translate-y`)

- **Secondary кнопка** — `.secondary-button`  
  - Белый фон, индиговая граница, мягкая тень
  - При hover — светло‑индиговый фон и усиленная граница

- **Transparent / ghost кнопка** — `.transparent-button`  
  - Прозрачный фон, серый текст
  - При hover — светло‑серый фон

- **Success и Danger кнопки** — `.success-button`, `.danger-button`  
  - Success: зелёный градиент (emerald/teal)
  - Danger: розово‑красный градиент (rose/pink)

**Рекомендация:**

- При генерации новых форм/страниц:
  - Главное действие — `class="primary-button"`.
  - Второстепенное — `class="secondary-button"` или `transparent-button`.
  - Успешные/опасные действия — соответствующие `success-button` / `danger-button`.

## Карточки, контейнеры и эффекты

- **Карточки / панели**:
  - Использовать `.box-shadow` для стандартных блоков:
    - Белый фон (в тёмной теме тёмно‑серый),
    - мягкая тень,
    - скругление `rounded-2xl`.
  - Для эффектов наведения — комбинировать с `.card-hover`.

- **Glass эффект**  
  - Класс `.glass` — полупрозрачный фон `bg-white/80` + `backdrop-blur-xl` (в тёмной теме тёмный).

- **Тонкая граница**  
  - `.subtle-border` — тонкая светлая/тёмная рамка.

- **Фокус**  
  - `.focus-ring` — стандартное кольцо фокуса через Tailwind `focus:ring-*`.

**Рекомендация:**

- Новые блоки/виджеты в админке строить как:
  - `<div class="box-shadow card-hover ...">...</div>`

## Статусы и бейджи

Стили для статусов находятся в `app.css`:

- Общий базовый класс для статусов:
  - `.label-pending`, `.label-processing`, `.label-closed`, `.label-canceled`, `.label-info`, `.label-fraud`, `.label-pending_payment`, `.label-completed`, `.label-active`
  - Маленький текст, белый цвет, `rounded-full`, градиентный фон.

- Раскраска по типу статуса:
  - Pending / pending_payment — жёлто‑оранжевые градиенты.
  - Processing — голубой‑синий градиент.
  - Completed / active — зелёный градиент.
  - Closed — сине‑фиолетовый градиент.
  - Canceled / fraud — красно‑розовый градиент.
  - Info — серый градиент.

- Текстовые статусы:
  - `.status-enable` — зелёный текст.
  - `.status-disable` — красный текст.

**Рекомендация:**

- Для любых статусных бейджей:
  - Использовать существующие классы `.label-*`.
  - Не придумывать новые схемы статусов без необходимости.

## Иконки

- Иконочный шрифт: `bagisto-admin`, классы `icon-*`, определены в `app.css`.
- Примеры: `icon-dashboard`, `icon-store`, `icon-notification`, `icon-exit`, и т.д.
- Иконки по умолчанию серые (`#6b7280`), в тёмной теме — светло‑серые.

**Рекомендация:**

- Для иконок всегда использовать уже заданные классы `icon-*` вместо SVG, если это иконка из набора Bagisto.
- Для однородности расположения — добавлять utility‑классы Tailwind (`text-xl`, `sm:text-2xl` и т.п.).

---

## Фон и темы (light/dark)

- Тема зависит от класса `html.dark` (есть переключатель в header).
- Фон страницы:
  - В светлой теме — сложный градиент из светлых серо‑слейтовых оттенков.
  - В тёмной теме — градиенты из тёмно‑серых/чёрных (`from-gray-950 via-slate-950` и т.п.).

**Рекомендация:**

- Новые компоненты должны выглядеть корректно как в светлой, так и в тёмной теме:
  - Учитывать `dark:`‑варианты (`dark:bg-gray-900`, `dark:text-gray-200`, `dark:border-gray-800` и т.д.).

## Layout и общая композиция

- Главный layout: `x-admin::layouts` (`layouts/index.blade.php`).
- Хедер:
  - `header` с фиксированным верхом, полупрозрачным фоном, лёгким blur, тенью.
  - Внутри —:
    - бургер‑меню,
    - логотип,
    - глобальный поиск,
    - кнопки (тема, переход в магазин, уведомления, AI, профиль администратора).

- Контент:
  - Основной контейнер имеет отступы `pt-5 px-4 sm:px-6 lg:pt-6 lg:px-8` и смещение под сайдбар.

**Рекомендация:**

- При добавлении новых страниц:
  - Всегда использовать `<x-admin::layouts>` и вписываться в существующую структуру,
  - Адаптация под мобильные/десктопные брейкпоинты через существующие `sm`, `md`, `lg`, `xl`.

## Использование этим правилом

Нейросетевой редактор кода, генерируя новый UI для админки, должен:

1. **Использовать токены и стили из `tailwind.config.js` и `app.css`.**
2. **Для кнопок, карточек, статусов и иконок — опираться на уже существующие CSS‑классы**, а не изобретать новые без необходимости.
3. **Соблюдать поддержку светлой и тёмной темы** (использовать `dark:`‑модификаторы).
4. **Сохранять визуальную последовательность**: градиенты, мягкие тени, округлые углы, аккуратные фокусы и hover‑эффекты.

Это правило должно подключаться, когда задача связана с:
- генерацией/рефакторингом Blade‑шаблонов админ‑панели,
- генерацией Vue/JSX‑разметки для админских компонентов,
- изменением UI Magic AI/AI‑ассистента внутри админки.
